# Контракт API: Сканер → Бэкенд

Устройство «Сканер» отправляет на бэкенд одно и то же событие в обоих режимах
(рисование на экране / фото рисунка с бумаги) — один и тот же формат запроса.

## Endpoint

```
POST /api/fish
Content-Type: application/json
```

Адрес сервера настраивается прямо в приложении сканера (поле на главном экране),
менять код не нужно — просто впишите реальный адрес бэкенда.

## Тело запроса

```json
{
  "fishType": "clownfish",
  "mode": "draw",
  "image": "data:image/png;base64,iVBORw0KGgo...",
  "createdAt": "2026-07-08T06:07:56.318Z"
}
```

| Поле        | Тип    | Значения / описание                                                                 |
|-------------|--------|--------------------------------------------------------------------------------------|
| `fishType`  | string | `clownfish`, `shark`, `octopus`, `whale`, `squid`, `crab`, `dolphin`, `turtle`, `fish` |
| `mode`      | string | `draw` (раскраска на экране) или `scan` (фото с камеры)                              |
| `image`     | string | base64 data URL, всегда **PNG с прозрачным фоном** (в обоих режимах). В scan сканер отправляет исходное фото с камеры (JPEG), бэкенд сам извлекает рыбку и убирает фон — остаётся только сама нарисованная рыбка на прозрачном фоне, готовая для наложения на сцену в Unity |
| `createdAt` | string | ISO 8601 timestamp                                                                    |

## Ответ сервера

Успех:
```json
{ "ok": true, "fishType": "clownfish", "mode": "draw" }
```
HTTP 200.

Ошибка:
```json
{ "ok": false, "error": "текст ошибки" }
```
HTTP 400/500 — сканер покажет пользователю кнопку "Попробовать снова" и повторит тот же запрос.

## CORS

Сканер — веб-страница, открытая с другого устройства (телефона), поэтому бэкенду
нужно разрешить кросс-доменные запросы:
```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: POST, OPTIONS
Access-Control-Allow-Headers: Content-Type
```
и отвечать `204` на `OPTIONS` (preflight).

## Размер запроса

Фото с камеры (после авто-обрезки) обычно 10-20 КБ в base64, но лимит на всякий
случай стоит поставить около **20 МБ** на body — на случай крупных/некачественно
сжатых кадров.

## Рабочий пример для проверки

Рабочая референс-реализация (то, на чём я тестировал сканер) лежит в
[animagia-mock-server/server.js](../animagia-mock-server/server.js) — простой
Node.js HTTP-сервер без зависимостей, принимает `/api/fish`, сохраняет картинки
в `received/`. Можно взять как отправную точку и заменить логику сохранения на
реальную (очередь, БД, передача на Unity-стену и т.д.).
